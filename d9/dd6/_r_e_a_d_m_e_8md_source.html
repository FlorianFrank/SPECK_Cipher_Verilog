1 # Speck Cipher Veriog implementation
2 
3 This repository contains a sample implementation of the SPECK64/128 cipher.
4 See [Reference](https://csrc.nist.gov/csrc/media/events/lightweight-cryptography-workshop-2015/documents/papers/session1-shors-paper.pdf).
5 
6 The cipher is using only ARX (Addition, Rotation and XOR) operations, which can be implemented fast in hardware as well as fast in hardware.
7 The cipher can be implemented using two repeating blocks, the key schedule as well as the round function.
8 Like visualized in the figure below.
9 
10 ![Structure](https://raw.githubusercontent.com/FlorianFrank/SPECK_Cipher_Verilog/main/documentation/figures/SPECKStructure.svg)
11 
12 The verilog implementation follows this scheme and is subdivided into three verilog modules.
13 
14 ### 1. Round function

19 
20 The round function is implemented as a state machine.
21 In the following figure the different states of the SPECK verilog implementation can be seen.
22 
23 ![Structure](https://raw.githubusercontent.com/FlorianFrank/SPECK_Cipher_Verilog/main/documentation/figures/round_function_module.png)
24 
25 #### State:
26 
27 - 0000: Wait for start signal to start the round function
28 - 0001: Assign p0 and p1 to dedicated registers
29 - 0010: Calculate right shift (>> 8) on p1
30 - 0011: Calculate p0 + p1
31 - 0100: Calculate left shift (<< 3) on p0 and p1 XOR subkey k0
32 - 0101: Calculate p0 = p0 + p1
33 - 0110: Assign result to ciphertext wire and indicate round finished by setting finished to high

43 The key schedule implementation is also part of a dedicated verilog module.
44 It is implemented as state machine similar to the round function.
45 
46 ![Structure](https://raw.githubusercontent.com/FlorianFrank/SPECK_Cipher_Verilog/main/documentation/figures/key_schedule_module.png)
47 
48 #### State:
49 
50 - 0000: Wait for start signal to start the key schedule function
51 - 0001: Assign k0 and k1 to dedicated registers
52 - 0010: Calculate right shift (>> 8) on k1
53 - 0011: Calculate k1 + k2
54 - 0100: calculate k1 = k1 XOR vector and left shift of k0 (<< 3)
55 - 0101: Calculate XOR of k1 and k2
56 - 0110: Assign result to new key indicate key schedule finished by setting finished wire to high
57 
58 ### Cipher control module
59 
60 This module is responsible to control the round function and key schedule modules
61 to execute three rounds of the SPECK64/128 cipher.
62 It sets the signal_start signals and waits for a round function or key schedule to be finished.
63 
64 ![Structure](https://raw.githubusercontent.com/FlorianFrank/SPECK_Cipher_Verilog/main/documentation/figures/SPECK_3_Rounds.png)
65 
66 The simulation above shows the execution of three rounds of the SPECK cipher.
67 The start_states wire shows the timing when each of the round functions is started.
68 Index 0 is assigned to round 1, 1 to round 2 and so on.
69 After executing the three rounds. The finish signal is set to high and the new ciphertext is assigned.
